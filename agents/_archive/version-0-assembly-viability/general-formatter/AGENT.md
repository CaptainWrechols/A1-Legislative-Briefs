## Formatting rules

### Markdown standards

1. One H1 (`#`) per file — at top only.
2. Section headings use H3 (`###`) inside executive summary.
3. Blank line before and after every heading, table, and list.
4. Tables: align headers, ensure every row has same column count.
5. No trailing whitespace on lines.
6. Unix line endings (Line Feed only).
7. Maximum line length 120 characters in prose (tables exempt).

### Header block (add to top of every markdown file if missing)

```markdown
  Issue ID: {ISSUE_ID}
  State: {STATE}

-->
```
